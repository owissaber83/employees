# 🎓 تسليم عمل الأكاديمية — استكمال من حيث انتهينا

> **لكلود على الجهاز الآخر:** اقرأ هذا الملف أولاً قبل أي تعديل على الأكاديمية.

## ⛔ القاعدة الأهم (لا تكسر الصفحة)
`public/academy.html` **مُولَّد آلياً (مترجَم مسبقاً)** — **لا تعدّله يدوياً أبداً**.

- **المصدر الوحيد للتعديل:** `academy.src.html` (في جذر المستودع، غير منشور — لا يخدمه Firebase).
- بعد أي تعديل على المصدر، **أعد البناء**:
  ```bash
  cd scripts && npm install && node build-academy.mjs
  ```
  ثم `git add academy.src.html public/academy.html` وcommit وpush.

**لماذا؟** الأكاديمية تطبيق React. أزلنا من المتصفح ترجمة JSX (Babel) وبناء CSS (Tailwind CDN) لتسريع التحميل، فصار الناتج JS ثابت (`React.createElement` classic) + Tailwind ساكن مضمّن. سكربت البناء يعيد إنتاج ذلك حرفياً (وضع classic إلزامي — التلقائي `jsx-runtime` يكسر الصفحة).

## 🏗️ المعمار
- React 18 UMD + ReactDOM من CDN (يبقيان).
- كل الحالة في `localStorage` — **لا يوجد باكند** (لا مصادقة/دفع).
- المنصّة **مجانية**: أُزيلت كل الأسعار وأسماء المدربين والاشتراك وتسجيل الدخول.
- الأصناف الديناميكية في Tailwind = صفر (كلها ثابتة) فالبناء الساكن آمن.

## 🎯 المهمة الحالية: الإثراء الاحترافي لبقية الدورات
المطلوب: تقمّص **أستاذ جامعي بخبرة 20 عاماً** في المجال، وإثراء **محتوى كل درس** ليخرج المحاسب الجديد جاهزاً لسوق العمل.

**نمط الإضافة لكل درس** (يُلحق بنهاية محتوى الدرس): شرح أعمق ➜ **قيود محاسبية** ➜ **مثال رقمي** ➜ `💼 من واقع الخبرة` ➜ `⚠️ أخطاء شائعة` ➜ **أسئلة المقابلات/سوق العمل**.

استخدم أصناف الصناديق الجاهزة: `content-box` · `example-box` · `formula-box` · `tip-box` · `warning-box`، والجداول `<table>`، و`<h3>/<h4>`.

### كيف تُضيف لدرس (تقنياً)
كل درس كائن فيه `content: \`...HTML...\``. **الحِق** أقسامك الجديدة **قبل الباكتيك المغلق** مباشرة:
- ابحث في `academy.src.html` عن نهاية محتوى الدرس (آخر `</div>` أو `</ul>` قبل الباكتيك)، وطابقها كـ `old_string` واجعل `new_string` = نفس النهاية + HTML الجديد + الباكتيك.
- **لا تغيّر عناوين الدروس** (`title`) — الاختبارات مرتبطة بها كمفتاح (`LESSON_QUIZZES`).
- تجنّب داخل المحتوى: الباكتيك `` ` `` و`${` (لأنها داخل template literal).
- عرّب أي مصطلح عقود إنجليزي (مثل ما فعلنا: «المقطوعية (Lump Sum)»)، لكن أبقِ أسماء الدوال/المعايير الرسمية إنجليزية (SUM/VLOOKUP/IFRS 15/GAAP…).

## ✅ ما أُنجز (لا تُعده)
منصّة كاملة: 8 دورات · اختبار لكل درس · **تقدّم + نقاط + سلسلة أيام + شارات + شهادة** · **بحث شامل بفتح مباشر للدرس** · **لوحة «مسيرتي»** · **ملاحظات** · **حفظ الدروس (Bookmarks)** · **طباعة الدرس PDF** · **الوضع الفاتح/الداكن** · ربط «طبّق الآن» بالـERP · أداء محسّن (بلا Babel/CDN).

**الإثراء الاحترافي المُنجز:**
- ✅ **محاسبة المقاولات** — الدروس الـ12 كاملة + **تبويب «الاختبار الشامل» بمراحل** (4 مراحل/28 سؤالاً) + توسيع أسئلة الدروس إلى 4 لكل درس.
- ✅ **المحاسبة العامة** — الدورة المحاسبية الكاملة (8 دروس: اليومية → الأستاذ → الميزان → التسويات → القوائم الأربع → النسب).
- ✅ **الضرائب والزكاة والفوترة** (`tax`) — الدروس الـ7 كاملة (تعمّق + قيود + مثال + من واقع الخبرة + أخطاء شائعة + أسئلة مقابلات).
- ✅ **IFRS** (`ifrs`) — الدروس الـ8 كاملة (نفس نمط الإثراء: GAAP، OCI، المقابل المتغيّر، المؤجّر، إعادة التقييم، CGU/العكس، تصنيف الأدوات، مصفوفة المخصصات).
- ✅ **CMA** (`cma`) — الدروس الـ10 كاملة (Part 1+2). ⚠️ **انتبه:** محتوى دروس CMA بعلامات **مفردة** `content: '...'` لا باكتيك — فلا تُدخِل `'` داخل المحتوى.

## 🔍 تحقّق قبل كل رفع
1. `node build-academy.mjs` ينجح ويطبع «✅ تم البناء» (عدد createElement > 100).
2. لا `import`/`jsx-runtime` في الناتج (السكربت يفشل تلقائياً إن وُجد).
3. صياغة سليمة: استخرج السكربت المضمّن ونفّذ `node --check`.
4. لا مصطلحات عقود إنجليزية عارية في الأسئلة.
