# 🔒 اختبارات قواعد الأمان (Firebase Security Rules Tests)

تتحقق هذه الاختبارات آليًا من أخطر ضمان في النظام: **عزل بيانات كل عميل (مستأجر) عن الآخر**، إضافةً إلى الأدوار وبوابة الاشتراك. شغّلها قبل أي تعديل على `database.rules.json` أو قبل عرض النظام على العملاء.

## المتطلبات (لمرة واحدة)
1. **Java** (يتطلبه محاكي Firebase): تأكد من وجوده بـ `java -version`.
2. **firebase-tools**: `npm install -g firebase-tools` (أو استخدم `npx firebase`).
3. تثبيت الحزم: من جذر المشروع نفّذ **`npm install`** (يُثبّت `@firebase/rules-unit-testing` و`firebase`).

## التشغيل
```bash
npm run test:rules
```
يشغّل الأمر محاكي قاعدة البيانات محليًا، ينفّذ الاختبارات، ثم يُغلق المحاكي. النتيجة المتوقعة: **كل الاختبارات ناجحة** (`N ناجح · 0 فاشل`). أي فشل = ثغرة محتملة في القواعد يجب معالجتها فورًا.

## ما الذي يُختبَر
- **العزل**: عضو المستأجر A لا يقرأ/يكتب بيانات المستأجر B، ولا يرى قائمة كل المستأجرين.
- **المصادقة**: غير المسجّل لا يقرأ/يكتب شيئًا؛ المستخدم المصادَق غير العضو محجوب.
- **الأدوار**: المحاسب يقرأ/يكتب القيود؛ المشاهد (viewer) لا يكتب؛ سجل التدقيق للمدير فقط.
- **الاشتراك**: المستأجر منتهي الاشتراك (`meta/accessUntil` في الماضي) لا يستطيع الكتابة.
- **المشغّل (operator)**: يستطيع قراءة قائمة المستأجرين.

## إضافة اختبارات
عدّل `tests/rules.test.mjs` — كل اختبار سطر واحد يلفّ عملية بـ `assertFails(...)` (يجب أن تُرفَض) أو `assertSucceeds(...)` (يجب أن تنجح).
